WORK PACKAGE 05: JSON-LD schema output

Goal:
- Output Article + Person + Organization schema.org structured data in `wp_head` using the same normalized author data that powers feed output. Multi-author support in schema (array of Person objects).

Primary targets:
- `inc/schema.php`
- `tests/phpunit/test-schema.php`

Depends on:
- WP-01 (adapter interface and normalized author data).

Problem statement:
- Google's E-E-A-T framework rewards structured author identity. WordPress schema output via Yoast or Rank Math handles single authors but multi-author attribution is poorly represented. A plugin that generates multi-author Article + Person schema from the same data source as feed output provides coherent identity across both channels.

Cross-cutting identity rule:
- WP-05 may consume `ap_actor_url` when it is available from WP-04/WP-05 adapter resolution.
- `ap_actor_url` is distinct from declared `profiles[]` links and from fediverse handles.
- `did:web:` is not part of WP-05 and should not be added to the active schema roadmap yet.

Planned changes:

1. Output JSON-LD on singular post pages (`inc/schema.php`):
   - Hook `wp_head` to output a `<script type="application/ld+json">` block.
   - Only on singular post views for supported post types.
   - Build Article schema with `author` as an array of Person objects.

2. Person object structure:
   ```json
   {
     "@type": "Person",
     "name": "Jane Doe",
     "url": "https://example.com/author/jdoe",
     "description": "Staff writer covering technology.",
     "image": "https://example.com/avatars/jdoe.jpg",
     "sameAs": [
       "https://mastodon.social/@jdoe",
       "https://github.com/jdoe"
     ]
   }
   ```
   - `name` from `display_name`.
   - `url` from author `url` field, falling back to `get_author_posts_url()` for WP users.
   - `description` from author `description` field.
   - `image` from `avatar_url`.
   - `sameAs` populated from `profiles` array (same data as Byline `byline:profile`).
   - `sameAs` may also include `ap_actor_url` when that field is present and confidently resolved.
   - Apply `byline_feed_schema_person` filter per Person object.

3. Article schema:
   ```json
   {
     "@context": "https://schema.org",
     "@type": "Article",
     "headline": "Post Title",
     "datePublished": "2026-03-07T12:00:00+00:00",
     "dateModified": "2026-03-07T14:00:00+00:00",
     "author": [ ...Person objects... ],
     "publisher": {
       "@type": "Organization",
       "name": "Site Name",
       "url": "https://example.com",
       "logo": { "@type": "ImageObject", "url": "..." }
     }
   }
   ```

4. Schema plugin compatibility detection:
   - Check if Yoast SEO or Rank Math is active.
   - If detected, offer a compatibility mode (controlled by a setting or filter) that either:
     - Disables this plugin's schema output entirely (let the SEO plugin handle it).
     - Extends the SEO plugin's schema via their respective filters to add multi-author Person data.
   - Default: if a known SEO plugin is active, disable own schema output and log a notice suggesting the filter approach.
   - Filter: `byline_feed_schema_enabled` — returns bool, default true unless SEO plugin detected.

5. Write PHPUnit tests (`tests/phpunit/test-schema.php`):
   - Single author: JSON-LD contains Article with single Person in author array.
   - Multiple authors: JSON-LD contains Article with ordered Person array.
   - Person fields populated correctly from normalized author data.
   - `sameAs` populated from profiles array.
   - `ap_actor_url` added to `sameAs` only when present, and never inferred from arbitrary profile URLs.
   - Guest authors without URLs: `url` field omitted.
   - Non-singular pages: no JSON-LD output.
   - SEO plugin detected: schema output disabled by default.
   - Filter overrides enable/disable.

Validation:
- Singular post pages include `<script type="application/ld+json">` with valid Article schema.
- Google Rich Results Test (search.google.com/test/rich-results) shows correct Article + Person data.
- Schema.org validator passes.
- Multi-author posts show array of Person objects.
- Yoast/Rank Math detection works correctly.
- PHPUnit suite green.
- PHPCS clean.

Notes:
- The `publisher` Organization should use `get_bloginfo( 'name' )` and site icon. A future enhancement could allow site-level Organization configuration matching Byline's `byline:org`.
- The `sameAs` array is the JSON-LD equivalent of Byline's `byline:profile` elements. Same data, different output format. This is the single-source-of-truth principle in action.
- `ap_actor_url` belongs here only as an ActivityPub identity URL. `did:web:` remains a later vision item and should not be added opportunistically to `sameAs` during WP-05.
- This work package can be developed in parallel with WP-02, WP-03, and WP-04 after WP-01 is complete.
